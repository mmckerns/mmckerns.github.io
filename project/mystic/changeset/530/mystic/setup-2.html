     44"""------------------------------------------------------ 
     45mystic: a simple model-independent inversion framework 
     46------------------------------------------------------ 
     47 
     48The mystic framework provides a collection of optimization algorithms 
     49and tools that allows the user to more robustly (and readily) solve 
     50optimization problems. All optimization algorithms included in mystic 
     51provide workflow at the fitting layer, not just access to the algorithms 
     52as function calls. Mystic gives the user fine-grained power to both 
     53monitor and steer optimizations as the fit processes are running. 
     54 
     55Where possible, mystic optimizers share a common interface, and thus can 
     56be easily swapped without the user having to write any new code. Mystic 
     57solvers all conform to a solver API, thus also have common method calls 
     58to configure and launch an optimization job. For more details, see 
     59`mystic.abstract_solver`. The API also makes it easy to bind a favorite 
     603rd party solver into the mystic framework. 
     61 
     62By providing a robust interface designed to allow the user to easily 
     63configure and control solvers, mystic reduces the barrier to implementing 
     64a target fitting problem as stable code. Thus the user can focus on 
     65building their physical models, and not spend time hacking together an 
     66interface to optimization code. 
     67 
     68Mystic is in the early development stages, and any user feedback is 
     69highly appreciated. Contact Mike McKerns [mmckerns at caltech dot edu] 
     70with comments, suggestions, and any bugs you may find.  A list of known 
     71issues is maintained at http://dev.danse.us/trac/mystic/query. 
     72 
     73 
     74Major Features 
     75============== 
     76 
     77Mystic provides a stock set of configurable, controllable solvers with:: 
     78 
     79    - a common interface 
     80    - the ability to impose solver-independent bounds constraints 
     81    - the ability to apply solver-independent monitors 
     82    - the ability to configure solver-independent termination conditions 
     83    - a control handler yielding: [pause, continue, exit, and user_callback] 
     84    - ease in selecting initial conditions: [initial_guess, random] 
     85    - ease in selecting mutation strategies (for differential evolution) 
     86 
     87To get up and running quickly, mystic also provides infrastructure to:: 
     88 
     89    - easily generate a fit model (several example models are included) 
     90    - configure and auto-generate a cost function from a model 
     91    - extend fit jobs to parallel & distributed resources 
     92    - couple models with optimization parameter constraints [COMING SOON] 
     93

     114Installation 
     115============ 
     116 
     117Mystic is packaged to install from source, so you must 
     118download the tarball, unzip, and run the installer:: 
     119 
     120    [download] 
     121    $ tar -xvzf mystic-%(thisver)s.tgz 
     122    $ cd mystic-%(thisver)s 
     123    $ python setup py build 
     124    $ python setup py install 
     125 
     126You will be warned of any missing dependencies and/or settings 
     127after you run the "build" step above. Mystic depends on numpy 
     128and sympy, so you should install them first. There are several 
     129functions within mystic where scipy is used if it is available; 
     130however, scipy is an optional dependency. Having matplotlib installed 
     131is necessary for running several of the examples, and you should 
     132probably go get it even though it's not required. Matplotlib is

     165Usage Notes 
     166=========== 
     167 
     168Probably the best way to get started is to look at a few of the 
     169examples provided within mystic. See `mystic.examples` for a 
     170set of scripts that demonstrate the configuration and launching of  
     171optimization jobs for one of the sample models in `mystic.models`. 
     172Many of the included examples are standard optimization test problems. 
     173 
     174Instructions on building a new model are in `mystic.models.abstract_model`. 
     175Mystic provides base classes for two types of models:: 
     176 
     177    - AbstractFunction   [evaluates f(x) for given evaluation points x] 
     178    - AbstractModel      [generates f(x,p) for given coefficients p] 
     179 
     180It is, however, not necessary to use the base classes in your own model. 
     181Mystic also provides some convienence functions to help you build a 
     182model instance and a cost function instance on-the-fly. For more 
     183information, see `mystic.mystic.forward_model`. 
     184 
     185All mystic solvers are highly configurable, and provide a robust set of 
     186methods to help customize the solver for your particular optimization 
     187problem. For each solver, a minimal interface is also provided for users 
     188who prefer to configure their solvers in a single function call. For more 
     189information, see `mystic.mystic.abstract_solver` for the solver API, and 
     190each of the individual solvers for their minimal (non-API compliant) 
     191interface. 
     192 
     193Mystic extends the solver API to parallel computing by providing a solver 
     194class that utilizes the parallel map-reduce algorithm. Mystic includes 
     195a set of defaults in `mystic.mystic.python_map` that mirror the behavior 
     196of serial python and the built-in python map function. Mystic solvers 
     197built on map-reduce can utilize the distributed and parallel tools provided 
     198by the `pathos` package, and thus with little new code solvers are 
     199extended to high-performance computing. For more information, see 
     200`mystic.mystic.abstract_map_solver`, `mystic.mystic.abstract_nested_solver`, 
     201and the pathos documentation at http://dev.danse.us/trac/pathos. 
     202 
     203Important classes and functions are found here:: 
     204 
     205    - mystic.mystic.solvers                [solver optimization algorithms] 
     206    - mystic.mystic.termination            [solver termination conditions] 
     207    - mystic.mystic.strategy               [solver population mutation strategies] 
     208    - mystic.mystic.monitors               [optimization monitors] 
     209    - mystic.mystic.tools                  [function wrappers, etc] 
     210    - mystic.mystic.forward_model          [cost function generator] 
     211    - mystic.models                        [a collection of standard models] 
     212    - mystic.math                          [some mathematical functions and tools] 
     213

     215

     219    - mystic.models.abstract_model         [the model API definition]
