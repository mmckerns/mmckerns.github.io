8solvers: minimal and expanded interfaces for optimization algorithms
9
10
11Standard Interface
12==================
13
14All of mystic's optimizers derive from the solver API, which provides
15each optimizer with a standard, but highly-customizable interface.
16A description of the solver API is found in `mystic.models.abstract_model`,
17and in each derived optimizer.  Mystic's optimizers are::
18    == Global Optimizers ==
19    DifferentialEvolutionSolver  -- Differential Evolution algorithm
20    DifferentialEvolutionSolver2 -- Price & Storn's Differential Evolution
21    == Pseudo-Global Optimizers ==
22    BuckshotSolver               -- Uniform Random Distribution of N Solvers
23    LatticeSolver                -- Distribution of N Solvers on a Regular Grid
24    == Local-Search Optimizers ==
25    NelderMeadSimplexSolver      -- Nelder-Mead Simplex algorithm
26    PowellDirectionalSolver      -- Powell's (modified) Level Set algorithm
27
28
29Minimal Interface
30=================
31
32Most of mystic's optimizers can be called from a minimal (i.e. one-line)
33interface. The collection of arguments is often unique to the optimizer,
34and if the underlying solver derives from a third-party package, the
35original interface is reproduced. Minimal interfaces to these optimizers
36are provided::
37    == Global Optimizers ==
38    diffev      -- DifferentialEvolutionSolver
39    diffev2     -- DifferentialEvolutionSolver2
40    == Pseudo-Global Optimizers ==
41    buckshot    -- BuckshotSolver
42    lattice     -- LatticeSolver
43    == Local-Search Optimizers ==
44    fmin        -- NelderMeadSimplexSolver
45    fmin_powell -- PowellDirectionalSolver
46
47
48More Information
49================
50
51For more information, please see the solver documentation found here::
52    - mystic.mystic.differential_evolution   [differential evolution solvers]
53    - mystic.mystic.scipy_optimize           [scipy local-search solvers]
54    - mystic.mystic.ensemble                 [pseudo-global solvers]
55
56or the API documentation found here::
57    - mystic.mystic.abstract_solver          [the solver API definition]
58    - mystic.mystic.abstract_map_solver      [the parallel solver API]
59    - mystic.mystic.abstract_ensemble_solver [the ensemble solver API]
60
61

80def LoadSolver(filename=None, **kwds):
81    """load solver state from a restart file"""
82    if filename is None:
83        filename = kwds['_state'] if '_state' in kwds else None
84    #XXX: only allow a list override keys (lookup values from self)
85#   if filename is None: filename = self._state
86#   if filename is None:
87#       solver = self
88#   else:
89    import dill
90    if filename: f = file(filename, 'rb')
91    else: return
92    try:
93        solver = dill.load(f)
94        _locals = {}
95        _locals['solver'] = solver
96        code = "from mystic.solvers import %s;" % solver._type
97        code += "self = %s(solver.nDim);" % solver._type
98        code = compile(code, '<string>', 'exec')
99        exec code in _locals
100        self = _locals['self']
101    finally:
102        f.close()
103    # transfer state from solver to self, allowing overrides
104    self._AbstractSolver__load_state(solver, **kwds)
105    self._state = filename
106    self._stepmon.info('LOADED("%s")' % filename)
107    return self
