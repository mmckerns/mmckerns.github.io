8# Abstract Ensemble Solver Class
9"""
10This module contains the base class for launching several mystic solvers
11instances -- utilizing a parallel "map" function to enable parallel
12computing.  This module describes the ensemble solver interface.  As with
13the AbstractSolver, the "Solve" method must be overwritten with the derived
14solver's optimization algorithm. Similar to AbstractMapSolver, a call to
15self.map is required.  In many cases, a minimal function call interface for a
16derived solver is provided along with the derived class.  See the following
17for an example.
18
19The default map API settings are provided within mystic, while
20distributed and high-performance computing mappers and launchers
21can be obtained within the "pathos" package, found here::
22    - http://dev.danse.us/trac/pathos
23
24
25Usage
26=====
27
28A typical call to a 'ensemble' solver will roughly follow this example:
29
30    >>> # the function to be minimized and the initial values
31    >>> from mystic.models import rosen
32    >>> lb = [0.0, 0.0, 0.0]
33    >>> ub = [2.0, 2.0, 2.0]
34    >>>
35    >>> # get monitors and termination condition objects
36    >>> from mystic.monitors import Monitor
37    >>> stepmon = Monitor()
38    >>> from mystic.termination import CandidateRelativeTolerance as CRT
39    >>>
40    >>> # select the parallel launch configuration
41    >>> from pyina.launchers import Mpi as Pool
42    >>> NNODES = 4
43    >>> nbins = [4,4,4]
44    >>>
45    >>> # instantiate and configure the solver
46    >>> from mystic.solvers import NelderMeadSimplexSolver
47    >>> from mystic.solvers import LatticeSolver
48    >>> solver = LatticeSolver(len(nbins), nbins)
49    >>> solver.SetNestedSolver(NelderMeadSimplexSolver)
50    >>> solver.SetStrictRanges(lb, ub)
51    >>> solver.SetMapper(Pool(NNODES).map)
52    >>> solver.SetGenerationMonitor(stepmon)
53    >>> solver.SetTermination(CRT())
54    >>> solver.Solve(rosen)
55    >>>
56    >>> # obtain the solution
57    >>> solution = solver.Solution()
58
59
60Handler
61=======
62
63All solvers packaged with mystic include a signal handler that
64provides the following options::
65    sol: Print current best solution.
66    cont: Continue calculation.
67    call: Executes sigint_callback, if provided.
68    exit: Exits with current best solution.
69
70Handlers are enabled with the 'enable_signal_handler' method,
71and are configured through the solver's 'Solve' method.  Handlers
72trigger when a signal interrupt (usually, Ctrl-C) is given while
73the solver is running.  ***NOTE: The handler currently is disabled
74when the solver has been launched in parallel.***
75
76"""
