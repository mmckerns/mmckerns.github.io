Pyre is an integration framework for high performance computing. Written in Python, pyre is an extensible, object-oriented framework for specifying and staging complex, multi-physics simulations. Pyre provides developers of scientific computing software with a set of development tools, such as the easy specification of user inputs for computing engines, a decomposition scheme to construct sophisticated scientific software in a layered architecture, and a mechanism for distributed debugging support. Pyre promotes intelligent software development by facilitating componentization, reusability, and the application of software design patterns.


If a developer girdles their existing code in a thin framework layer, the code becomes a pyre component, and is available for deployment in larger framework applications. Pyre manage the life-cycle and interactions of components in an elegant way, and removes the burden from the user in writing potentially fragile application infrastructure. By enforcing component interface specifications, pyre attempts to solve a very hard problem -- to make independently developed applications work together seamlessly. Because pyre components act as building blocks for larger applications, and pyre applications can be configured with simple key-value pairs, pyre developers benefit from much shorter concept-to-deployment costs.

M.A.G. Aivazis, et al., “pyre: an integration framework for high-performance computing”, http://danse.us/trac/pyre
